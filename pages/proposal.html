Technical Documentation Proposal

Project title:

Proposal for the Development of the “Sharpen” Mobile Application (Mockup)

Tyler Moore

Technical Writer

Date: January 2026

Executive Summary

This proposal outlines the development of Sharpen, a mobile application designed to provide users with a mobile prompt generator. The app seeks to give ideas for designers and those in STEM fields through a seamless, user-centered experience.

Background/Problem Statement

Designers and STEM students are improving their skills for future jobs and advancement within their respective fields. They desire a challenge and are lacking inspiration for new projects to produce and display on their portfolios.

Presently, no web solution exists that generates creative ideas for branding, marketing, product/UX design, or inventions.

Objectives

  • Provide users with new ideas for creation in different design and STEM fields.

  • Generate over 100+ prompts for users.

  • Improve users’ portfolios.

Target Audience

  • Product/UX, Marketing, and Brand Designers

  • STEM professionals, educators, and students

  • Technical Skill Level: Low to moderate

Scope of Work

  • Documentation types to be created:

    • Responsive design requirements documentation

    • UX-focused user flow documentation

    • Feature descriptions and functional specifications

    • Release notes for initial launch

  • Excluded:

    • Backend API development

    • Marketing content

    • Ongoing web maintenance post-launch

Deliverables

  • Website Requirements Document

  • User Flow Diagrams

  • Feature Specification Document

  • Editable source files (HTML, CSS, Java, Markdown/PDF)

Documentation Approach/Methodology

  • Style guide

  • Agile documentation aligned with sprint cycles

  • Plain-language, task-based documentation

  • Two revision cycles per document

Tools & Technologies

  • Figma (wireframes and user flows)

  • Markdown & Google Docs

  • Jira (task tracking)

  • GitHub (version control)

Timeline & Milestones

  • Discovery & Research: 1 week

  • Documentation Drafting: 2 weeks

  • Review & Revisions: 1 week

  • Final Delivery: 1 week

Roles & Responsibilities

Technical Writer:

  • Create and maintain all documentation.

  • Coordinate review and revisions.

Stakeholders:

  • Provide access to APIs and requirements.

  • Review and approve documentation.

Assumptions & Constraints

  • Stakeholders are available for weekly check-ins

  • Transit APIs provide reliable real-time data

  • The project timeline is fixed at 5 weeks

Success Metrics

  • On-time delivery of documentation

  • Stakeholder approval with minimal revisions

  • Improved clarity of app requirements

  • Reduced development rework due to clear documentation

Conclusion/Next Steps

Sharpen solves roadblocks for creators by generating prompts by providing access to a concise, user-centered, responsive website. Upon approval, project discovery can begin.
